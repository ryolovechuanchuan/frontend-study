# 📱 Day52 - Responsive Web Design Practice

## 🇹🇼 中文

### 📖 專案介紹

這是 **180 天前端工程師學習計畫的 Day52**。

本專案主要練習響應式網頁設計（Responsive Web Design），透過 Flexbox 與 Media Query，讓 Navbar、Hero、Card 區塊與 Footer 能夠根據不同螢幕尺寸自動調整版面。

本次練習包含桌機版、平板版與手機版，並理解 Desktop First、Mobile First、Breakpoint 與 CSS 覆蓋規則。

### 🚀 功能與版面

- 響應式 Navbar
- 響應式 Hero Section
- Hero 按鈕與 Hover 效果
- 響應式 Card Layout
- Flexbox 自動換行
- Footer 區塊
- 桌機、平板與手機版面
- Media Query 樣式切換
- 卡片自動調整寬度
- 避免手機版水平捲軸

### 💡 學習心得

透過本次練習，我理解了響應式設計並不是一套全新的排版方式，而是在不同螢幕尺寸下，使用 Media Query 覆蓋原本的 CSS。

我也學會先完成桌機版，再透過 `max-width` 調整手機版，以及使用 `min-width` 建立 Mobile First 的版面。透過 Navbar、Hero 與 Card Layout 的實作，我開始能夠根據螢幕寬度調整排列方向、卡片寬度與內容間距。

## 🇺🇸 English

### 📖 Project Description

This project is part of my **180-Day Front-End Developer Roadmap — Day 52**.

The main goal of this project is to practice Responsive Web Design by combining Flexbox and Media Queries. The navigation bar, hero section, card layout, and footer automatically adjust according to different screen sizes.

The project includes desktop, tablet, and mobile layouts while exploring Desktop First, Mobile First, breakpoints, and CSS override behavior.

### 🚀 Features

- Responsive navigation bar
- Responsive hero section
- Hero button with hover interaction
- Responsive card layout
- Automatic Flexbox wrapping
- Footer section
- Desktop, tablet, and mobile layouts
- Media Query style switching
- Flexible card widths
- Prevention of horizontal scrolling on mobile

### 🛠️ Technologies

- HTML5
- CSS3
- Flexbox
- Media Queries
- Responsive Web Design

### 📚 What I Learned

- `@media`
- `max-width`
- `min-width`
- Breakpoints
- Desktop First
- Mobile First
- CSS Cascade
- CSS Overrides
- `flex-direction`
- `flex-wrap`
- `flex-grow`
- `flex-shrink`
- `flex-basis`
- `gap`
- `width: 100%`
- `flex: 1 1 250px`
- Responsive navigation bars
- Responsive card layouts

### 💡 Reflection

Through this project, I learned that responsive design is not a completely different layout system. It is the process of overriding existing CSS rules at different screen sizes using Media Queries.

I also practiced building a desktop layout first and adapting it for mobile devices with `max-width`, while learning how `min-width` is used in a Mobile First approach. By implementing the navigation bar, hero section, and card layout, I became more comfortable changing layout direction, widths, spacing, and alignment across different devices.

## 🇯🇵 日本語

### 📖 プロジェクト概要

このプロジェクトは、**180日間フロントエンド学習計画の Day52** として作成しました。

Flexbox と Media Query を利用し、ナビゲーションバー、Hero セクション、カードレイアウト、Footer が画面サイズに応じて自動的に変化するレスポンシブWebデザインを実装しました。

デスクトップ、タブレット、スマートフォン向けのレイアウトを作成し、Desktop First、Mobile First、ブレークポイント、CSS の上書きルールについて学習しました。

### 🚀 主な機能

- レスポンシブナビゲーションバー
- レスポンシブ Hero セクション
- Hero ボタンとホバーエフェクト
- レスポンシブカードレイアウト
- Flexbox による自動折り返し
- Footer セクション
- デスクトップ、タブレット、スマートフォン対応
- Media Query によるスタイル切り替え
- カード幅の自動調整
- スマートフォンでの横スクロール防止

### 🛠️ 使用技術

- HTML5

- Responsive Web Design

### 💡 学んだこと

今回の練習を通して、レスポンシブデザインは新しいレイアウト技術ではなく、Media Query を利用して画面サイズごとに既存の CSS を上書きする考え方であることを理解しました。

また、最初にデスクトップ版を作成し、`max-width` を利用してスマートフォン版へ調整する Desktop First の方法と、`min-width` を利用する Mobile First の考え方を学びました。Navbar、Hero、Card Layout の実装を通して、画面幅に応じて配置方向、カード幅、余白を調整できるようになりました。
